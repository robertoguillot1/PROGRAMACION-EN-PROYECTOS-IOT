01 · Definición del Proyecto

¿Qué es un sistema de
riego hidropónico IoT?

Un sistema autónomo que usa sensores para tomar decisiones de riego sin intervención humana, accesible desde cualquier dispositivo conectado a la red.

La hidroponía es una técnica de cultivo que permite crecer plantas sin suelo, usando soluciones nutritivas en agua. Al integrarlo con IoT (Internet of Things), logramos un sistema capaz de monitorear en tiempo real las condiciones del cultivo — temperatura, humedad del ambiente y humedad del sustrato — y actuar automáticamente sobre una bomba de agua para mantener condiciones óptimas de crecimiento.

— Definición del sistema basada en el código ESP32 del proyecto

El ESP32 actúa como el cerebro del sistema: lee los sensores cada 2 segundos, verifica las condiciones de humedad cada 3 segundos, y mantiene la bomba encendida durante 10 segundos cuando el suelo está seco. Además, sirve una interfaz web que permite el control remoto desde cualquier navegador.

02 · Componentes del Sistema

Hardware utilizado

Cada componente cumple una función específica en el ecosistema de monitoreo y control del riego.

ESP32 DevKit C v4

Microcontrolador principal. Gestiona sensores, controla la bomba, sirve el servidor web y gestiona la conexión WiFi simultáneamente.

WiFi · GPIO · ADC · I2C · 240MHz

Sensor DHT22

Sensor digital de temperatura y humedad ambiental de alta precisión. Se conecta al pin GPIO 4 del ESP32.

PIN: GPIO 4 · ±0.5°C · ±2-5% RH

Sensor Humedad Suelo

Potenciómetro deslizante que simula el sensor capacitivo de humedad del sustrato. Mapea 0–3300mV a 0–100% de humedad.

PIN: GPIO 35 · ADC · 0-3.3V

Módulo Relay

Controla la bomba de agua de manera aislada. El ESP32 envía señal LOW al pin 13 para activar el riego.

PIN: GPIO 13 · LOW=ON · HIGH=OFF

LCD 20×4 I2C

Pantalla de 4 líneas que muestra temperatura, humedad ambiente, humedad del suelo y estado de la bomba en tiempo real.

SDA: GPIO 21 · SCL: GPIO 22 · 0x27

Servidor Web ESP32

Servidor HTTP en puerto 80 que sirve una interfaz de monitoreo con auto-refresh cada 5 segundos y endpoint de control de bomba.

Puerto 80 · /toggle · Refresh 5s

Condición de Riego

La bomba se ACTIVA automáticamente cuando la humedad del suelo cae por debajo del 30%. Se mantiene durante 10 segundos y luego se detiene.

Condición de Paro

La bomba se DETIENE cuando la humedad del suelo alcanza el 70%, evitando el exceso de agua y el daño por inundación del sistema radicular.

05 · Beneficios del Proyecto

¿Por qué automatizar el riego?

El sistema ofrece ventajas clave sobre el riego manual tradicional.

90%

Ahorro de Agua

El riego solo se activa cuando es necesario según el sensor, evitando desperdicios de agua comparado con el riego por cronómetro fijo.

24h

Monitoreo Continuo

El ESP32 opera de forma ininterrumpida, verificando las condiciones del cultivo y actuando en tiempo real sin necesidad de presencia humana.

100%

Control Remoto

El servidor web integrado permite monitorear y controlar el sistema desde cualquier dispositivo en la misma red WiFi.

IoT

Escalable y Conectado

La arquitectura basada en ESP32 permite integración futura con plataformas cloud como AWS IoT, Firebase o MQTT para análisis de datos históricos.
